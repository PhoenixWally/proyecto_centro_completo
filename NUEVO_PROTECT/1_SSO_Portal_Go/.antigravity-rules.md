# CONTEXTO DEL PROYECTO: Sentinel Appliance (Bare-Metal Windows)

# REGLAS ESTRICTAS DE ENTORNO:

- Sistema Host: Windows Server.
- PROHIBIDO usar Docker, contenedores o docker-compose.
- PROHIBIDO generar configuraciones para Linux (ni fstab, ni systemd).
- Los ejecutables (.exe) funcionarán como Servicios de Windows nativos.
- Enrutamiento: Path-Based bajo un solo dominio (ej. `/sentinel/`, `/analitico/`). Nginx gestiona SSL manual y ruteo frontal.

# ESTADO DE LA BASE DE DATOS (POSTGRESQL - YA CREADA)

Usa este esquema exacto para cualquier query (No generes script de creación):

- `roles` (id, nombre, nivel_poder)
- `provincias` (id UUID, nombre)
- `usuarios` (id UUID, username, password_hash, rol_id)
- `usuario_provincias` (usuario_id, provincia_id)
- `estaciones` (id UUID, nombre, provincia_id, ip_red)
- `aplicaciones` (id UUID, nombre, ruta_base)
- `usuario_aplicaciones` (usuario_id, aplicacion_id)

# REGLAS DE DESARROLLO GO (Portal SSO)

- Backend en `127.0.0.1:8080`.
- Usar `pgx` para conectar a PostgreSQL.
- Hash de contraseñas exclusivo con Argon2.
- Manejo de sesiones con JWT en HttpOnly Cookies.

# REGLAS DE DESARROLLO C++ (Motor Sentinel)

- Backend en `127.0.0.1:8081`.
- Lectura EXCLUSIVA de directorios locales usando `std::filesystem` de C++17.
- PROHIBIDO el uso de APIs de red Windows (`WNetAddConnection2A`, `winnetwk.h`, `Mpr.lib`).
